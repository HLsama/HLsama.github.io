消融效果

2026/3/13 技术 UI

给到的需求是消融效果，它在游戏中非常常用，是个很常见的效果。本文将先介绍消融shader的使用方式，再去讲一下它的实现原理。

unity版本：Unity6000.062f1

1.使用方式

其实消融shader的使用并不复杂，基本手动将各个参数调整一下就能学会使用，但参数数量还算是有些多，所以还是挨个介绍一下。

  • DissolveColor

很简单，指的就是消融边界的颜色，支持HDR，需要注意的是，为了让unity的后处理生效，需要将Canvas的RenderMode的模式设置为ScreenSpace—Camera。

  • NoiseTex

消融效果都需要噪声贴图，选一个就好，不同噪声的话边缘的消融形状也不一样，下面是个噪声范例：

  • NoiseOffestSpeed

消融的边缘会流动，这个参数控制的是流动速度

  • Angle

Angle控制的是消融开始的方向，角度制，0度时方向是从最左边开始消融

  • DissolveRange

它控制的是消融过程本身，随着数值增大，会按照某个方向消融。

  • NoiseSoft

NoiseSoft控制的是纯色边缘所占据边缘的比例，0时完全没有纯色边缘，随着NoiseSoft值增大，纯色边缘所占据的越来越多

  • NoiseRange

NoiseRange指的是消融残留部分的范围，值越小残留越少，消融边界越接近一条直线

  • SrcOffest

SrcOffest指的是消融边缘图像本身的扰动，值越大，消融边缘的位置被扭曲的越严重。

2.消融原理

最简单的消融的话非常好实现，直接采样噪声图，然后跟给个值进透明剪切即可。这样的消融会根据整个噪声的明暗进行消融，白色的地方最先消融，黑色的地方后消融。如果再加上个值，跟原先的消融进度做个区分的话，就可以将消融的边缘拆分开来，乘上个颜色就是消融边缘颜色，而加上的这个区分值就是控制消融边界比例的值。

而这次的消融制作在其基础上加了一些东西。主要是消融的方向性与边界残留的控制，要让消融从某个方向到另一个方向，这需要计算出一个遮罩：

float rad = radians(_Angle);
float2 dir = float2(cos(rad), sin(rad));
float2 pos = uv-float2(0.5,0.5) ;
float posOffset = dot(normalize(dir), pos)*0.5+0.5;
_DissolveRange = remap(_DissolveRange, 0, 1, -_NoiseRange, 1 + _NoiseRange);
float fromDir = _DissolveRange - _NoiseRange;
float alphaDir = remap(posOffset, fromDir, _DissolveRange, 0, 1);
alphaDir = saturate(alphaDir);

2.1消融方向

在拿到消融的角度_Angle后，先从角度制度转为弧度制，再拆成正弦函数和余弦函数的组合，用于后续的角度计算。对于旋转，由于UV的原点（0，0）右下角，直接进行旋转的话，旋转中心在UV原点的位置，这不是我们想要的，所以我们需要给UV-0.5，将它的原点移动到图片的中心。对于图片上的任意一点的UV值，我们都可以看作是一个以图片中心为原点的向量值，将其与旋转方向做点乘，我们就可以知道哪些像素需要被影响。此外，这个值是-1~1的值，我们将其*0.5+0.5给映射到0~1的区间。

下面是posOffset值的可视化：

其从左到右就是0~1的值，也就代表着消融的进度。

2.2重映射函数remap

为了控制消融进度以及残留，我们需要对0~1的消融进度重映射，而其中使用的函数则是remap函数，它可以将指定区间的值映射到另一个区间。它并不包含在HLSL的库中，需要自己构建，它长成这个样子：

half remap(half x, half t1, half t2, half s1, half s2)
{
    return (x - t1) / (t2 - t1) * (s2 - s1) + s1;
}

其中x是参考值，t1t2是想要映射的区间，s1s2是映射到的区间。

写成式子是这样：y = (x - t1) / (t2 - t1) * (s2 - s1) + s1。

我们先解析前半部分： y = (x - t1) / (t2 - t1)。

假设 t2 = 0.7， t1 = - 0.8 代入至公式中，可得：

y = (x - (-0.8)) / (0.7 - (-0.8))，从代入式可知：

当 x = -0.8 时，y = 0；

当 x = 0.7 时，y = 1；

作图如下：

根据求取的结果与示意图，我们可得出如下结论：

当x = t1时，y = 0；

当x = t2时，y = 1；

前半部分的计算我们已经分析完毕了，其实这部分算法也是remap算法，因为它已经能让我们达到重映射的目的了。接下来我们结合整个公式进行分析，分析方式还是以代入数字的方式进行。

对 y = (x - t1) / (t2 - t1) * (s2 - s1) + s1 公式进行数字代入。

假设 t2 = 0.7，t1 = -0.8保持不变，令s2 = 0.3，s1 = 0.2待入至公式得：

y = (x - (-0.8)) / (0.7 - (-0.8)) * (0.3 - 0.2) + 0.2。

根据代入结果，作图如下：

可得结果：

当 x = -0.8时，y = 0.2；

当 x = 0.7 时， y = 0.3；

也就是，

当 x = t1 时，y = s1；

当 x = t2 时，y = s2；

2.3边界残留的控制以及原图扰动

对于原消融扰动控制，我们先从0~1给映射到-_NoiseRange~1 + _NoiseRange区间，然后我们计算一个fromDir值，它为映射后的 _DissolveRange- _NoiseRange值。这样它的取值范围就是- _NoiseRange~2* _NoiseRange。然后我我们再将之前计算的01的posOffset的fromDirDissolveRange区间映射到0~1，就得到了一个新的值，它在完全消除到NoiseRange之间会有个自己控制的NoiseRange过度，可以控制残留下来区间值。

可视化如下：

最后是对原图的扰动并不是什么难事，其实原理就是采样噪声后在把噪声值加到采样原图的UV上，最后给个参数控制强度罢了。

相关代码如下：

float2 NoiseUV= uv +_NoiseOffestSpeed *dir* _Time.x;
half dissolveTexAlpha = tex2D(_NoiseTex, NoiseUV).r;
half reverseAlphaDir = 1 - alphaDir;

half2 uvDiff = uv + half2(0, reverseAlphaDir * dissolveTexAlpha * _SrcOffest);
half4 color = IN.color * tex2D(_MainTex, uvDiff);
