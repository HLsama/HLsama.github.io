渐变背景的原理与使用方法

2026/3/8 技术 UI

渐变背景其实是个很简单的东西，所以本文更偏向这个shader的功能介绍，大概一边稍微讲一下它的原理，一边就顺带着将使用方式讲清楚了，理解原理的话使用起来也更得心应手吧。

unity版本：Unity6000.062f1

1.基本的渐变

给到的需求是丝滑变化颜色，需要让美术指定要变化的颜色。其实有两种思路，其一是在材质面板上暴露多个颜色参数，然后不断lerp。其二是让美术直接做一张渐变图，然后让时间沿着UV的U轴不断变化，从而达到颜色随着时间变化的目的。第一种方式弊端太多，例如我们不能实时去控制材质面板中颜色的数量，这样美术就不能自由控制整体的变化规律，其次调整起来参数复杂，对美术来说也不够直观。而第二种方式则方便太多，让美术自己在Photoshop中指定渐变，这样渐变的规律，颜色都可以牢牢的把控在自己手里。

1.1渐变原理

首先展示下渐变图：

这是一个256*1的渐变图，由于UV的V轴完全不需要变化，对于整个屏幕来说颜色是完全一致的，所以制作的渐变图可以在Y轴的方向只存在一个像素，可以节约贴图大小等成本，而渐变图的两段像素一致则可以保证在不断循环采样的过中颜色不跳变。而具体采样方式如下：

shader：

half4 color = SAMPLE_TEXTURE2D(_BaseMap, sampler_BaseMap, float2(frac(_Time.x*_Speed),0.5))；

可以看到对于这张贴图的采样，由于在V轴方向是不需要变化的，所以V轴是固定的值0.5，也可以防止V值采样在边缘以至于出现意料之外的情况。而U轴采样则是一个自动流动的时间，乘以_Speed就可以控制时间的流速了。frac将数值控制在小数的范围内，防止随着时间的累计数值越来越大。

2.使用噪声

如果只有渐变可能导致背景颜色过于呆板。我们可以使用噪声，给背景加一点点变化。

2.1噪声

还是先看噪声吧： Substance_graph_output.tga

思路是让uv的u值配合时间函数使噪声流动起来，然后直接乘到渐变的纹理上。

shader

half Noise = SAMPLE_TEXTURE2D(_NoiseTex, sampler_NoiseTex, float2(IN.uv.x-_Time.x*_Speed,IN.uv.y)).r;
Noise = pow(Noise,_Pow);
half4 color = SAMPLE_TEXTURE2D(_BaseMap, sampler_BaseMap, float2(frac(_Time.x*_Speed),0.5)) * _BaseColor+Noise*_NoiseAlpha;

其中Pow函数是对Noise值进行_Pow次方计算，小数的值次方后会更小，所以可以控制噪声的软硬程度：
