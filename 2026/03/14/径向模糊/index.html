径向模糊

2026/3/14 技术 UI

本次的内容是径向模糊，它在表现加速度等场合非常常用，由于shader使用起来并不复杂，就直接介绍原理了。

unity版本：Unity6000.062f1

径向模糊原理

其实在原理介绍方面也没太多的点可说，这次就不分点了。

径向模糊的原理比较直接，首先选取一个模糊中心，然后对每个采样点的UV基于这个模糊中心进行偏移，并进行一定次数的迭代采样，最终将采样得到的RGB值累加，并除以迭代次数。

核心代码如下：

float2 dir = (float2(_X,_Y)-IN.texcoord)*_BlurRadius*0.01;

float4 color = 0;
for(int t=0;t<_Iteration;t++)
{
    color += tex2D(_MainTex,(IN.texcoord - 0.5) * _UVScale + 0.5);
    IN.texcoord+=dir;
}
color = color/_Iteration;

其中 _X与 _Y共同构成了偏移的中心，而每个像素的UV值，可以看作以UV原点为原点的向量，如果想要将偏移中心设置为图像正中心的话，就需要将X与Y的值设置为（0.5，0.5）。

__BlurRadius则是每次迭代时偏移的距离。然后我们迭代采样，每次迭代的时候朝着偏移值方向偏移一步，不断循环的话，就可以得到径向模糊了。当然，由于每个像素是不断累计叠加的，不做处理会让图像过曝，此时我们需要将最终的像素值除以循环次数，来让图像的值回到正常。此外，UI的画布大小是有限的，如果不做限制的话 ，就会粗暴的截断，导致穿帮。如图所示：

所以使用了 _UVScale值来调整UV在画布中所占的比例 ，以呈现完整的图像。

可以看到它类似于将图像复制了一份，然后移动，那么随着迭代次数增多，会有怎样的效果呢？

实际上就是在每次将图像复制一份的同时又朝向固定方向偏移了以下。需要注意的是，过多的迭代次数实际上就是多次采样贴图会导致性能的下降，实际使用的时候要考虑到这一点。
