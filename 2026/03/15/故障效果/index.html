故障效果

2026/3/15 技术 UI

故障效果并非是由单一内容组成的，本次内容主要是图像的随机跳动、扫描线和RGB色彩分离，他们实际上都是在图像的UV上做文章（其实所有UI的内容几乎都算是在UV上做文章，毕竟也没什么其他可供操作的数据），所以可以分别先处理UV并叠加，并最后拿来采样贴图，我将逐一讲解他们的原理。

unity版本：Unity6000.062f1

1.图像的随机跳动

先看效果：

很简单，其实就是图像沿着UV的U轴左右晃嘛，那么只需要给UV.x加上一个来回变动的值就好。

那么为了让这个晃动不那么呆板，我们需要一个随机数生成函数：

float randomNoise(float x, float y)
{
    return frac(sin(dot(float2(x, y), float2(12.9898, 78.233))) * 43758.5453);
}

对于UV的晃动，由于需要左右循环，所以把他放在了Sin函数中，并用frac控制了只取小数，这样它的值就是一个0~1的区间。

然后我们用时间函数 _Time去生成这个随机数，这样这个随机数就可以随着游戏的进行随机变化了。此外， _JitterIntensity可以控制这个随机数的大小，来控制跳动的变化范围。

相关代码如下：

float2 uv = IN.texcoord+float2(_JitterIntensity*randomNoise(_Time.y, _Time.x) ,0);

2.扫描线抖动故障

同样的，先看效果：

可以发现实际上就是让每一行的UV随机抖动，算是之前内容的进阶版。

首先对于随机函数，这次我们生成时将UV.y作为输入之一，这样相同横行的随机值将会是相同的，此外将输入的值从01映射到-11，代码如下：

float jitter = randomNoise(uv.y, _Time.x) * 2 - 1;

然后，我们还要控制抖动线条的数量，我们可以生成一个一个遮罩，只有0和1，这样就可以让一部分线条显示出来，一部分消失掉。至于生成的方式，我们可以对于任意jitter值我们取它的绝对值，然后用Step，这样所有低于某个_LineCount的值就都可以被抹平为0，这样就可以控制线条的数量了。我们将结果乘到原先的jitter上，然后再通过 _LineJitterIntensity来控制抖动的剧烈程度。相关代码如下：

jitter *= step(_LineCount, abs(jitter)) * _LineJitterIntensity;
uv += float2(jitter, 0);

3.RGB分离

效果：

实现起来并不复杂，只需要拿到不断变化的随机数，然后三次采样图像即可，每次采样分别只取图片的R、G、B通道，并且其中两个通道的UV加上彼此相反的UV值，这样每个通道就会单独以相反的方向抖动。需要注意的是，记得每次采样的时候都分别采样一下透明通道，来达到正确的透明效果，并且合并时透明通道的值可能超过1，需要手动给钳制一下。

相关代码如下：

float splitAmount = _RGBSeparationIntensity * randomNoise(_Time.y, 2);
float2 colorR = (IN.color * (tex2D(_MainTex, uv+float2(splitAmount,0)) + _TextureSampleAdd)).ra;
float2 colorG = (IN.color * (tex2D(_MainTex, uv) + _TextureSampleAdd)).ga;
float2 colorB = (IN.color * (tex2D(_MainTex, uv+float2(-splitAmount,0)) + _TextureSampleAdd)).ba;
float4 color = float4(colorR.r,colorG.r,colorB.r,colorR.g+colorG.g+colorB.g);
color.a = min(1.0,color.a);
