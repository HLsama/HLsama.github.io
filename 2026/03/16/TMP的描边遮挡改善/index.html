TMP的描边遮挡改善

2026/3/16 技术 UI

TMP全程TextMeshPro，是一款非常强大的文字系统，关于它的内容非常复杂，这次只将目光聚焦在如何解决描边遮挡问题，关于TMP更详细的解读将在其他文章中进行。

unity版本：Unity6000.062f1

1.原因解析

首先看TMP原先有什么问题。当具有描边的文字逐渐靠近到一起的时候，文字之间的描边会相互遮挡如图：

至于造成他们发生这样的情况的原因可以这样理解：TMP的每一个字母都可以看作是一个面片，而一行字则可以看作是一个都立的模型。而一个模型中的每个面片的渲染顺序遵循其内幕的网格ID，对于TMP而言，他们的网格就是按照字母顺序从前到后排列的。所以当第一个面片渲染后，第二个模型的一部分像素可能就会覆盖掉第一个模型，这就早成了他们之间的相互遮挡。由于他们同属于同一个模型，所以我们也不可能用遮罩之类的手段去识别文字本体和描边部分，只能另寻他法。

2.自定义的深度写入

untiy的着色器有个功能：我们可以忽略掉物体本身的深度值，而将每次模型的深度写成我们想要指定的值，他就是SV_Depth。

使用示例：

float4 frag(v2f i, out float depth : SV_Depth) : SV_Target
{
    float4 col = float4(1, 0, 0, 1);
    depth = 0.5;
    return col;
}

而TMP是基于SDF的渲染方式，每个文字都会有个距离场，越接近文字中心的值越大

如图所示：

而对于我们的需求来说，描边正是远离文字中心的部分。基于这一点，我们可以将这个距离场写入深度，并开启深度测试，这样文字本体（文字中心部分）的深度值就会永远大于描边部分（文字边缘部分），使深度测试通过，就会永远让文字本体通过深度测试了。
