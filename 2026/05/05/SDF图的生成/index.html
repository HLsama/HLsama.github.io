SDF图的生成

2026/5/5 技术 UI

之前讲了那么多的关于SDF的内容终于轮到SDF图的生成了，但是在这之前，请先去看下关于计算着色器的内容，此文将大量依赖它

unity版本：Unity6000.062f1

1.SDF简单介绍

SDF就是计算图中的各点到图形最近的距离。它是有方向的，所以图像外面的值是大于0的，图像的边缘的值为0，而图像的内测的值小于0。

下图就表示了SDF中各点之间的关系。

2.SDF图的计算方式

SDF方式有很多，这里列举其中三个，前两个并非工程中用到的就简单介绍，最后一个我会简单讲解。

2.1、暴力算法

基于上文SDF的思想，最简单的方式就是暴力计算，首先计算出各点到最近白色值的最近距离，然后计算各点到最近黑色位置的距离，最后相减，得到的就是SDF值。

下面是网上找到的相关代码：

import math

import cv2 as cv
import numpy as np

width = 100

min_dis = np.ones((width,width),dtype=np.uint8)*(width+1)
min_dis2 = np.ones((width,width),dtype=np.uint8)*(width+1)

ori_img = cv.imread('a.png',0)
show_img = cv.resize(ori_img, (width,width),interpolation=cv.INTER_AREA)

def getDis(x, y, x1, y1):
    return math.sqrt((x-x1)*(x-x1) + (y-y1)*(y-y1))

#白色区域，在表面外部
def isWhite(x,y):
    return show_img[x][y]==255

#各点到最近的白色距离
for x in range(width):
    for y in range(width):
        tmp_min_dis = width+1
        for x1 in range(width):
            for y1 in range(width):
                if(isWhite(x1,y1)):
                    tmp_min_dis = min(tmp_min_dis,getDis(x,y,x1,y1))
        min_dis[x][y] = tmp_min_dis

#各点到最近的黑色距离
for x in range(width):
    for y in range(width):
        tmp_min_dis = width+1
        for x1 in range(width):
            for y1 in range(width):
                if(not(isWhite(x1,y1))):
                    tmp_min_dis = min(tmp_min_dis,getDis(x,y,x1,y1))
        min_dis2[x][y] = tmp_min_dis

final_dis = min_dis2 - min_dis
min_val = np.min(final_dis)
max_val = np.max(final_dis)
val_dis = max_val-min_val
final_dis = (final_dis+min_val)*255/val_dis

cv.imshow("img",final_dis)
cv.waitKey(0)
cv.destroyAllWindows()

这个算法可以说是最拉跨的一个，生成速度很慢，一般完全不会有人用这样的方式。

2.2 、8ssedt算法

8ssedt是在CPU上运行的比较流行的算法，它可以在线性时间里通过二值化的黑白图来生成一张SDF。

其基本思路就是将一个复杂问题拆解成连续简单的问题。因为SDF记录的每个像素到图像边缘的距离，而这个距离是连续的，所以我们可以通过临近的像素推导出当前像素的距离。例如我们可以检索当前像素点上面的像素的SDF值，那么当前与上面的距离值为1，那么当前像素的SDF值就为周围的SDF值+1，同理，左上的话就是+根号2。

下面是网上找到的相关代码

#include "SDL/sdl.h"
#include <math.h>

#define WIDTH  256
#define HEIGHT 256

struct Point
{
    //dx,dy表示对于当前点的偏移值
    int dx, dy;

    int DistSq() const { return dx*dx + dy*dy; }
};

struct Grid
{
    Point grid[HEIGHT][WIDTH];
};

Point inside = { 0, 0 };
Point empty = { 9999, 9999 };
Grid grid1, grid2;

Point Get( Grid &g, int x, int y )
{
    // OPTIMIZATION: you can skip the edge check code if you make your grid 
    // have a 1-pixel gutter.
    if ( x >= 0 && y >= 0 && x < WIDTH && y < HEIGHT )
        return g.grid[y][x];
    else
        return empty;
}

void Put( Grid &g, int x, int y, const Point &p )
{
    g.grid[y][x] = p;
}

void Compare( Grid &g, Point &p, int x, int y, int offsetx, int offsety )
{
    //获取当前点偏移后的点
    Point other = Get( g, x+offsetx, y+offsety );
    //给获取的点的dx和dy设置对应的偏移值
    other.dx += offsetx;
    other.dy += offsety;

    if (other.DistSq() < p.DistSq())
        p = other;
}

// Now all we have to do is run the propagation algorithm. See the paper for exactly what's happening here, 
// but basically the idea is to see what the neighboring pixel has for it's dx/dy,
//  then try adding it onto ours to see if it's better than what we already have.
void GenerateSDF( Grid &g )
{
    // Pass 0
    //遍历当前点以及左右、左下、右下、正下方的点，找到距离最短的点存储在网格对应位置处
    for (int y=0;y<HEIGHT;y++)
    {
        for (int x=0;x<WIDTH;x++)
        {
            Point p = Get( g, x, y );
            Compare( g, p, x, y, -1,  0 );
            Compare( g, p, x, y,  0, -1 );
            Compare( g, p, x, y, -1, -1 );
            Compare( g, p, x, y,  1, -1 );
            Put( g, x, y, p );
        }

        for (int x=WIDTH-1;x>=0;x--)
        {
            Point p = Get( g, x, y );
            Compare( g, p, x, y, 1, 0 );
            Put( g, x, y, p );
        }
    }

    // Pass 1
    for (int y=HEIGHT-1;y>=0;y--)
    {
        for (int x=WIDTH-1;x>=0;x--)
        {
            Point p = Get( g, x, y );
            Compare( g, p, x, y,  1,  0 );
            Compare( g, p, x, y,  0,  1 );
            Compare( g, p, x, y, -1,  1 );
            Compare( g, p, x, y,  1,  1 );
            Put( g, x, y, p );
        }

        for (int x=0;x<WIDTH;x++)
        {
            Point p = Get( g, x, y );
            Compare( g, p, x, y, -1, 0 );
            Put( g, x, y, p );
        }
    }
}

int main( int argc, char* args[] )
{
    if ( SDL_Init( SDL_INIT_VIDEO ) == -1 )
        return 1;

    SDL_Surface *screen = SDL_SetVideoMode( WIDTH, HEIGHT, 32, SDL_SWSURFACE );
    if ( !screen )
        return 1;

    // Initialize the grid from the BMP file.
    SDL_Surface *temp = SDL_LoadBMP( "test.bmp" );
    temp = SDL_ConvertSurface( temp, screen->format, SDL_SWSURFACE ); 
    SDL_LockSurface( temp );
    for( int y=0;y<HEIGHT;y++ )
    {
        for ( int x=0;x<WIDTH;x++ )
        {
            Uint8 r,g,b;
            Uint32 *src = ( (Uint32 *)( (Uint8 *)temp->pixels + y*temp->pitch ) ) + x;
            SDL_GetRGB( *src, temp->format, &r, &g, &b );
            
            // Points inside get marked with a dx/dy of zero.
            // Points outside get marked with an infinitely large distance.
            // 两个网格，一个内部设置成0，外部设置成正无穷，另一个网格相反。
            if ( g < 128 )
            {
                Put( grid1, x, y, inside );
                Put( grid2, x, y, empty );
            } else {
                Put( grid2, x, y, inside );
                Put( grid1, x, y, empty );
            }
        }
    }
    SDL_UnlockSurface( temp );

    // Generate the SDF.
    GenerateSDF( grid1 );
    GenerateSDF( grid2 );
    
    // Render out the results.
    SDL_LockSurface( screen );
    for( int y=0;y<HEIGHT;y++ )
    {
        for ( int x=0;x<WIDTH;x++ )
        {
            // Calculate the actual distance from the dx/dy
            //计算偏移值的点距离当前点的距离
            int dist1 = (int)( sqrt( (double)Get( grid1, x, y ).DistSq() ) );
            int dist2 = (int)( sqrt( (double)Get( grid2, x, y ).DistSq() ) );
            int dist = dist1 - dist2;

            // Clamp and scale it, just for display purposes.
            int c = dist*3 + 128;
            if ( c < 0 ) c = 0;
            if ( c > 255 ) c = 255;

            Uint32 *dest = ( (Uint32 *)( (Uint8 *)screen->pixels + y*screen->pitch ) ) + x;
            *dest = SDL_MapRGB( screen->format, c, c, c );
        }
    }
    SDL_UnlockSurface( screen );
    SDL_Flip( screen );

    // Wait for a keypress
    SDL_Event event;
    while( true )
    {
        if ( SDL_PollEvent( &event ) ) 
        switch( event.type )
        {
        case SDL_QUIT:
        case SDL_KEYDOWN:
            return true;
        }
    }

    return 0;
}

如果只是CPU的话这个方式确实很快了，但是我们有GPU，别忘了我们的GPU最擅长图像这种大规模重复计算，下面这个算法就是利用GPU的SDF算法。

2.3、独立扫描 Saito Toiwaki 算法

2.3.1、原理

对于一个二维图像，我们进行两次变换，首先我们看第一个变换

变换1：我们对图像的横列进行两次扫描

扫描1

从左到右对于点（i，j）

如果点（i，j）为黑色点，我们就设置G（i，j）为0，并且记录或者更新这个黑色点的位置

如果点（i，j）为白色点，我们就设置G（i，j）为距离最近左侧黑色点的距离，并如果没黑色点则设置为图像的宽度值

扫描2

从右到左对于点（i，j）

如果点（i，j）为黑色点，我们就设置G（i，j）为0，并且记录或者更新这个黑色点的位置

如果点（i，j）为白色点，计算距离最近右侧黑色点的距离，并如果没黑色点则设置为图像的宽度值，并且与G（i，j）比较，如果小于G（i，j）的值，则更新G（i，j）为距离值

变换2：遍历图像的每个点，对于点 (i, j)

对于当前像素（i，j），计算它到所在列的每个点（x，j）的距离

将各点距离加上G（x，j）的值后，去其中的最小值，就为点（i，j）的距离值了。

其实这个算法在计算上是比8ssedt要慢的，但是它各个像素的计算相对独立，不太依赖于其他像素，所以可以利用GPU的并行性，同时运算。

2.3.2、代码部分

这个算法需要进行两次变换，所以对于计算着色器，我们需要两个核函数：

#pragma kernel Process0
#pragma kernel Process1

然后我们分别需要原图值Raw，第一阶段后的中间值DataProcess0，以及最后的输出值Result

对于第一个函数，我们设置每个线程组中的线程为[numthreads(1,4,1)]，这是因为考虑到了每个线程组的临时寄存器（数组）数量问题。

如果你的临时寄存器过多的话unity会提示：

warning X4714: sum of temp registers and indexable temp registers times 8 threads exceeds the recommended total 16384. Performance may be reduced at kernel Process0 (on d3d11)

也就是每个线程组中的临时寄存器最好不超过16384， 超出的话会引起性能下降，而实际上每个线程会处理一整行的像素，并存下它的临时数据，正反两次扫描，也就是需要2048个可引索临时寄存器，这样算下来，每个线程组只能有8个线程，再加上一些其他可能需要的临时寄存器，就会超标了，所以为了安全考虑，最好是[numthreads(1,4,1)]。

第一个核函数：

[numthreads(1,4,1)]
void Process0 (uint2 id : SV_DispatchThreadID)
{
    float2 sdpr[2048];
    uint size = _TexSize.x;
    float2 d = float2(999999,999999);
    float2 sqr_d = float2(0,0);

    for (uint i = 0; i < size; ++i)
    {
        float raw = Raw[uint2(i,id.y)].x;
        if (raw < 1)
        {
            d.x = 0;
            
            d.y += 1;
            sdpr[i] = float2(0,d.y * d.y);
        }
        else
        {
            d.x += 1;
            sdpr[i] = float2(d.x * d.x,0);
            
            d.y = 0;
        }
    }

    d = float2(999999,999999);
    for (uint j = size - 1; j > 0; --j)
    {
        float raw = Raw[uint2(j,id.y)].x;
        
        if (raw < 1)
        {
            d.x = 0;
            
            d.y += 1;
            sqr_d.y = d.y * d.y;
        }
        else
        {
            d.x += 1;
            sqr_d.x = d.x * d.x;
            
            d.y = 0;
        }
        DataProcess0[uint2(j,id.y)] = float2(min(sqr_d.x,sdpr[j].x), min(sqr_d.y,sdpr[j].y));
    }
}

第二个核函数就没有这种问题，使用的是[numthreads(32,32,1)]

第二个核函数：

[numthreads(32,32,1)]
void Process1 (uint2 id : SV_DispatchThreadID)
{
    float2 sdpo;
    uint size = _TexSize.y;
    float2 min_sqr_d = id.yy * id.yy + DataProcess0[uint2(id.x,0)];
    
    for (uint i = 1; i < size ; ++i)
    {
        uint2 dist = i - id.y;
        float2 sqr_d = dist * dist;
        sdpo = sqr_d + DataProcess0[uint2(id.x,i)];
        min_sqr_d = min(sdpo, min_sqr_d);
    }

    if (min_sqr_d.x > 0)
    {
        Result[id] = sqrt(min_sqr_d.x)/size + 0.5;
    }
    else
    {
        Result[id] = 0.5 - sqrt(min_sqr_d.y)/size;
    }
}

当然，实际情况和这个示例不同，例如对于SDF来说，实际上输出并不需要1024 * 1024，512 * 512就够了，或者可能会根据需要清理最外圈像素、指定输入图哪个通道才是源二值图等，这里就不过多介绍了。
